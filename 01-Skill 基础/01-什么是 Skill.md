# 01 - 什么是 Skill

## 📌 概述

**Skill（技能）** 是 OpenClaw 中的核心扩展机制，用于增强 AI 助手的特定能力。

想象一下：你希望 AI 助手能够查询天气、管理待办事项、操作文件等。Skill 就是为这些特定任务而设计的可插拔模块。

## 🎯 Skill 解决什么问题

### 问题 1：能力扩展

```
┌─────────────────────────────────────┐
│          基础 AI 助手                 │
│  - 通用对话                          │
│  - 知识问答                          │
│  - 文本生成                          │
│                                     │
│  ❌ 无法访问外部数据                  │
│  ❌ 无法执行特定操作                  │
└─────────────────────────────────────┘
```

### Skill 的解决方案

```
┌─────────────────────────────────────┐
│          基础 AI 助手                 │
└─────────────────┬───────────────────┘
                  │
    ┌─────────────┼─────────────┐
    │             │             │
┌───▼───┐   ┌────▼────┐   ┌───▼───┐
│天气 Skill│ │待办 Skill│ │文件 Skill│
└───────┘   └─────────┘   └───────┘
     │             │             │
┌────▼────┐  ┌─────▼────┐  ┌────▼────┐
│天气 API  │  │本地存储   │  │文件系统  │
└─────────┘  └──────────┘  └─────────┘
```

## 🔑 核心概念

### 1. Skill 包

一个 Skill 是一个独立的目录，包含：
- **SKILL.md**: Skill 的描述和配置
- **代码文件**: 实现具体功能
- **依赖**: package.json 等

### 2. SKILL.md

Skill 的"身份证"，定义：
- Skill 名称和描述
- 触发条件
- 可用工具
- 对话流程

### 3. 工具函数

Skill 提供的具体能力：
```javascript
// 示例：查询天气
async function getWeather(city) {
  const response = await fetch(`https://api.weather.com/${city}`);
  return response.json();
}
```

### 4. 对话流程

定义 AI 如何与用户交互：
```
用户：北京天气怎么样？
  ↓
Skill 识别意图
  ↓
调用天气查询工具
  ↓
返回结果并格式化
  ↓
AI: 北京今天晴，25°C...
```

## 💡 Skill 的特点

### 1. 模块化

每个 Skill 专注于一个领域：
```
✅ weather-skill (天气查询)
✅ todo-skill (任务管理)
✅ file-skill (文件操作)

❌ all-in-one-skill (什么都做)
```

### 2. 可插拔

可以随时安装/卸载 Skill：
```bash
# 安装天气 Skill
openclaw skills install weather-skill

# 卸载
openclaw skills remove weather-skill
```

### 3. 可组合

多个 Skill 可以协同工作：
```
用户：查询北京天气并保存到待办
  ↓
天气 Skill → 查询天气
  ↓
待办 Skill → 创建提醒
```

### 4. 可扩展

基于现有 Skill 扩展：
```
基础天气 Skill
  ↓
扩展：添加空气质量
  ↓
扩展：添加穿衣建议
```

## 🆚 Skill vs 其他方案

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| **Skill** | 标准化、易分享、可组合 | 需要学习规范 | AI 能力扩展 |
| **插件** | 功能强大 | 平台锁定 | 特定平台扩展 |
| **脚本** | 灵活、直接 | 难以管理 | 一次性任务 |
| **API** | 通用、灵活 | 需要集成工作 | 服务间通信 |

## 🌟 Skill 的优势

1. **标准化**: 统一的格式和规范
2. **易分享**: 可以发布到 Skill Hub
3. **易管理**: 统一的安装/更新机制
4. **可测试**: 内置测试框架
5. **文档化**: SKILL.md 提供完整文档

## 🚀 快速体验

### 查看已安装的 Skill

```bash
# 列出所有 Skill
openclaw skills list

# 查看 Skill 详情
openclaw skills info weather-skill
```

### 使用 Skill

```
用户：北京天气怎么样？

AI: [自动调用天气 Skill]
    北京今天晴，气温 25°C，
    空气质量良好，适合户外活动。
```

### 安装新 Skill

```bash
# 从 Skill Hub 安装
openclaw skills install todo-skill

# 从本地安装
openclaw skills install ./my-skill
```

## 📦 Skill 包结构

### 基础结构

```
my-skill/
├── SKILL.md              # Skill 描述（必需）
├── index.js              # 入口文件
├── tools/                # 工具函数
│   ├── weather.js
│   └── utils.js
├── package.json          # 依赖管理
└── README.md             # 使用说明
```

### SKILL.md 示例

```markdown
# Skill 名称
天气查询 Skill

# 描述
查询全球城市天气信息

# 触发词
天气、气温、天气预报

# 工具
- getWeather(city): 查询城市天气
- getAirQuality(city): 查询空气质量

# 示例对话
用户：北京天气怎么样？
AI: 北京今天晴，25°C...
```

## 🎯 适用场景

### ✅ 适合用 Skill

- **外部 API 集成**: 天气、新闻、股票
- **本地操作**: 文件管理、系统控制
- **业务流程**: 审批、报告生成
- **专业知识**: 医疗咨询、法律咨询

### ❌ 不适合用 Skill

- **简单计算**: 直接用 AI 即可
- **通用知识**: AI 已具备的能力
- **一次性脚本**: 不需要复用
