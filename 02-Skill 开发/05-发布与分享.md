## 🚀 发布流程

### 1. 发布到 GitHub

```bash
# 初始化 Git
git init

# 添加文件
git add .

# 提交
git commit -m "Initial commit: weather-skill v1.0.0"

# 创建 GitHub 仓库并推送
git remote add origin https://github.com/yourname/weather-skill.git
git push -u origin main
```

### 2. 创建 Git 标签

```bash
# 创建版本标签
git tag -a v1.0.0 -m "Release version 1.0.0"

# 推送标签
git push origin v1.0.0
```

### 3. 发布到 Skill Hub

```bash
# 登录 Skill Hub
openclaw hub login

# 发布 Skill
openclaw skills publish

# 或者指定目录
openclaw skills publish ./weather-skill
```

### 4. 验证发布

```bash
# 查看已发布的 Skill
openclaw hub list

# 查看 Skill 详情
openclaw hub info weather-skill

# 测试安装
openclaw skills install weather-skill
```

## 使用

### 查询当前天气

```
用户：北京天气怎么样？
AI: 北京今天晴，气温 25°C，空气质量良好。
```

### 查询天气预报

```
用户：上海未来 3 天天气
AI: 上海未来 3 天天气预报：
     今天：晴，25°C
     明天：多云，27°C
     后天：小雨，23°C
```

## 配置

| 配置项 | 说明 | 默认值 | 必需 |
|--------|------|--------|------|
| WEATHER_API_KEY | 天气 API 密钥 | 无 | 是 |

### 获取 API 密钥

1. 访问 https://weatherapi.com
2. 注册账号
3. 创建应用获取 API Key
4. 在 OpenClaw 配置中添加：
   ```bash
   openclaw config set WEATHER_API_KEY your-key
   ```

## 📊 版本管理

### 语义化版本

```
主版本号。次版本号。修订号
  ↑      ↑      ↑
  |      |      └─ 向后兼容的问题修正
  |      └──────── 向后兼容的功能新增
  └─────────────── 不兼容的 API 修改

示例：
1.0.0 - 初始版本
1.0.1 - 修复 bug
1.1.0 - 新增功能
2.0.0 - 不兼容更新
```

## 🔒 安全发布

```bash
# 检查是否有硬编码的密钥
